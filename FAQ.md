# ❓ Часто задаваемые вопросы (FAQ)

## Общие вопросы

### Что делает этот бот?

Бот защищает Telegram группы от ботов и ИИ, проверяя новых участников с помощью различных типов проверок (капча, математика, логика и т.д.).

### Нужен ли сервер для работы?

Нет! Бот полностью автономен и работает на твоём компьютере без внешних сервисов, баз данных или вебхуков.

### Сколько стоит использование?

Бот полностью бесплатный и с открытым исходным кодом.

### Можно ли использовать в нескольких группах?

Да! Один бот может работать в неограниченном количестве групп одновременно.

## Установка и настройка

### Какая версия Python нужна?

Python 3.8 или выше. Проверь версию: `python --version`

### Как получить токен бота?

1. Открой @BotFather в Telegram
2. Отправь `/newbot`
3. Следуй инструкциям
4. Скопируй полученный токен

### Где прописать токен?

Токен уже прописан в коде. Если нужен свой, замени в файле `bot.py` или `config.py`:
```python
BOT_TOKEN = "твой_токен_здесь"
```

### Как изменить настройки?

Открой файл `config.py` и измени нужные параметры:
- `max_attempts` - количество попыток
- `timeout` - время на проверку
- `enabled_types` - типы проверок

## Использование

### Как добавить бота в группу?

1. Найди бота в Telegram
2. Нажми "Добавить в группу"
3. Выбери группу
4. Дай права администратора

### Какие права нужны боту?

Обязательные права:
- ✅ Удаление сообщений
- ✅ Бан пользователей
- ✅ Ограничение пользователей
- ✅ Приглашение пользователей

### Как работает проверка?

1. Новый участник входит в группу
2. Бот ограничивает его права
3. Отправляет проверку (капча)
4. Участник проходит проверку
5. Бот снимает ограничения

### Сколько времени на проверку?

По умолчанию 5 минут. Можно изменить в настройках.

### Сколько попыток даётся?

По умолчанию 5 попыток. Можно изменить в настройках.

### Что если не пройти проверку?

Пользователь будет кикнут из группы и получит уведомление в личку.

## Проверки

### Какие типы проверок есть?

1. Эмодзи - найди правильный эмодзи
2. Математика - реши пример
3. Последовательность - продолжи ряд
4. Цвета - выбери цвет
5. Паттерн - найди повторяющийся
6. Логика - ответь на вопрос
7. Сравнение - сравни числа
8. Слова - найди животное
9. Счёт - посчитай эмодзи
10. Лишнее - найди лишний элемент

### Можно ли отключить некоторые типы?

Да! В файле `config.py` измени список `enabled_types`.

### Можно ли добавить свои проверки?

Да! Добавь новый метод в класс `ChallengeGenerator` в файле `bot.py`.

### Как изменить сложность?

В файле `config.py` измени параметр `difficulty`:
- `easy` - лёгкие проверки
- `medium` - средние (по умолчанию)
- `hard` - сложные

## Модерация

### Как забанить пользователя?

Команда: `/ban @username`

### Как разбанить?

Команда: `/unban @username`

### Как замутить пользователя?

Команда: `/mute @username [минуты]`

Пример: `/mute @user 60` - мут на 60 минут

### Как добавить в белый список?

Команда: `/whitelist @username`

Пользователи из белого списка не проходят проверку.

### Как добавить в чёрный список?

Команда: `/blacklist @username`

Пользователи из чёрного списка автоматически банятся.

## Команды

### Какие команды для всех?

- `/start` - Начать работу
- `/help` - Помощь
- `/verify` - Пройти проверку вручную
- `/status` - Статус верификации
- `/stats` - Статистика

### Какие команды для админов?

- `/ban` - Забанить
- `/unban` - Разбанить
- `/kick` - Кикнуть
- `/mute` - Замутить
- `/unmute` - Размутить
- `/warn` - Предупредить
- `/settings` - Настройки
- `/whitelist` - Белый список
- `/blacklist` - Чёрный список

### Как посмотреть статистику?

Команда: `/stats`

Показывает:
- Количество верифицированных
- Количество забаненных
- Процент успеха
- Использование команд

## Проблемы

### Бот не отвечает

Проверь:
1. Бот запущен?
2. Есть интернет?
3. Токен правильный?
4. Бот не заблокирован?

### Бот не может кикать

Проверь:
1. Бот - администратор?
2. Есть права на бан?
3. Бот выше пользователя в иерархии?

### Проверка не приходит

Проверь:
1. Бот может писать в группу?
2. Группа не в режиме "только админы"?
3. Бот не в чёрном списке?

### Ошибка "Invalid token"

Решение:
1. Проверь токен
2. Убери лишние пробелы
3. Скопируй токен заново

### Ошибка "No module named 'telegram'"

Решение:
```bash
pip install python-telegram-bot
```

### Бот кикает всех

Проверь:
1. Настройки в `config.py`
2. Белый список
3. Логи в `bot.log`

### Бот не кикает ботов

Это нормально! Бот пропускает других ботов (кроме проверки).

## Безопасность

### Безопасен ли бот?

Да! Бот:
- Не собирает личные данные
- Не отправляет данные на сервера
- Работает локально
- Открытый исходный код

### Где хранятся данные?

Все данные хранятся локально в файлах:
- `bot_data.json` - пользователи
- `stats.json` - статистика
- `bot.log` - логи

### Можно ли удалить данные?

Да! Просто удали файлы `bot_data.json` и `stats.json`.

### Как сделать бэкап?

Скопируй файлы:
- `bot_data.json`
- `stats.json`
- `config.py`

### Сколько групп может обслуживать?

Неограниченно! Зависит от мощности компьютера.

### Сколько пользователей может проверить?

Неограниченно! Бот обрабатывает проверки асинхронно.

## Расширение

### Можно ли добавить свои функции?

Да! Бот написан на Python с понятным кодом.

### Как добавить новый тип проверки?

1. Открой `bot.py`
2. Найди класс `ChallengeGenerator`
3. Добавь новый метод `generate_xxx_challenge()`
4. Добавь в список `challenge_types`

### Можно ли изменить сообщения?

Да! Все сообщения в файле `config.py` в словаре `MESSAGES`.

### Как добавить новый язык?

Создай копию словаря `MESSAGES` с переводом и добавь переключатель языка.

## Обновление

### Как обновить бота?

1. Скачай новую версию
2. Замени файлы (кроме `config.py`)
3. Перезапусти бота

### Сохранятся ли данные?

Да! Данные в `bot_data.json` и `stats.json` сохранятся.

### Как узнать версию?

В файле `config.py` смотри `BOT_VERSION`.
